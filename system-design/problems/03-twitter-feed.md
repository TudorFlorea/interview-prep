## 1. Requirements Gathering

### Functional Requirements

1. **Post tweets** - Create text posts (280 chars), with optional media
2. **Follow/Unfollow** - Follow other users to see their content
3. **Home timeline** - View personalized feed of tweets from followed users
4. **User profile/timeline** - View a user's own tweets
5. **Like and Retweet** - Engage with content
6. **Search** - Find users and tweets (optional for MVP)

### Non-Functional Requirements (FCC-SLEDS)

| Aspect | Requirement |
|--------|-------------|
| **Fault Tolerance** | High availability (99.99%), eventual consistency acceptable |
| **CAP** | AP - Availability over consistency |
| **Compliance** | Content moderation, GDPR |
| **Scalability** | 500M DAU, 500M tweets/day |
| **Latency** | Timeline load < 200ms, Tweet post < 500ms |
| **Environment** | Global, mobile-heavy traffic |
| **Durability** | Tweets must be durable |
| **Security** | Authentication, spam prevention |

## 3. Data Model

### Core Entities

```sql
-- Users
CREATE TABLE users (
    user_id BIGINT PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    display_name VARCHAR(100),
    bio TEXT,
    profile_image_url VARCHAR(500),
    followers_count INT DEFAULT 0,
    following_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Tweets
CREATE TABLE tweets (
    tweet_id BIGINT PRIMARY KEY,  -- Snowflake ID
    user_id BIGINT NOT NULL,
    content VARCHAR(280) NOT NULL,
    media_urls JSON,
    reply_to_tweet_id BIGINT,      -- NULL if not a reply
    retweet_of_id BIGINT,          -- NULL if not a retweet
    quote_tweet_id BIGINT,         -- NULL if not a quote
    like_count INT DEFAULT 0,
    retweet_count INT DEFAULT 0,
    reply_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    
    INDEX idx_user_id (user_id),
    INDEX idx_created_at (created_at)
);

-- Follows (Graph relationship)
CREATE TABLE follows (
    follower_id BIGINT NOT NULL,
    followee_id BIGINT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    
    PRIMARY KEY (follower_id, followee_id),
    INDEX idx_followee (followee_id)
);

-- Likes
CREATE TABLE likes (
    user_id BIGINT NOT NULL,
    tweet_id BIGINT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    
    PRIMARY KEY (user_id, tweet_id),
    INDEX idx_tweet_id (tweet_id)
);
```

```python
class TimelineService:
    def __init__(self, redis: Redis, tweet_db: Database):
        self.redis = redis
        self.tweet_db = tweet_db
        self.MAX_TIMELINE_SIZE = 800
        self.CELEBRITY_THRESHOLD = 10000
    
    async def get_timeline(self, user_id: int, page: int = 0, size: int = 20) -> List[Tweet]:
        # 1. Get pre-computed timeline (pushed tweets)
        start = page * size
        end = start + size - 1
        
        tweet_ids = await self.redis.zrevrange(
            f"timeline:{user_id}",
            start,
            end
        )
        
        # 2. Get celebrities this user follows
        celebrities = await self.get_followed_celebrities(user_id)
        
        # 3. Pull recent tweets from celebrities
        if celebrities:
            celebrity_tweets = await self.get_recent_tweets_from_users(celebrities)
            tweet_ids = self.merge_and_sort(tweet_ids, celebrity_tweets)[:size]
        
        # 4. Hydrate tweet IDs to full tweet objects
        tweets = await self.hydrate_tweets(tweet_ids)
        
        return tweets
    
    async def fanout_tweet(self, tweet: Tweet):
        """Push tweet to follower timelines"""
        
        # Check if celebrity
        follower_count = await self.get_follower_count(tweet.user_id)
        
        if follower_count > self.CELEBRITY_THRESHOLD:
            # Skip fanout for celebrities
            return
        
        # Get all followers
        followers = await self.get_followers(tweet.user_id)
        
        # Batch push to Redis
        pipeline = self.redis.pipeline()
        for follower_id in followers:
            pipeline.zadd(
                f"timeline:{follower_id}",
                {tweet.id: tweet.created_at.timestamp()}
            )
            # Trim to keep timeline size bounded
            pipeline.zremrangebyrank(
                f"timeline:{follower_id}",
                0,
                -self.MAX_TIMELINE_SIZE - 1
            )
        
        await pipeline.execute()
```

## 12. Trending Topics

```python
class TrendingService:
    def __init__(self, redis: Redis):
        self.redis = redis
        self.WINDOW_SECONDS = 3600  # 1 hour window
        self.BUCKETS = 60  # 1-minute buckets
    
    async def record_hashtag(self, hashtag: str):
        """Record hashtag occurrence in current time bucket"""
        bucket = int(time.time() // 60)  # Current minute
        key = f"trending:{bucket}"
        
        # Increment count for this hashtag in this bucket
        await self.redis.zincrby(key, 1, hashtag)
        await self.redis.expire(key, self.WINDOW_SECONDS)
    
    async def get_trending(self, limit: int = 10) -> List[Tuple[str, int]]:
        """Get top trending hashtags from last hour"""
        current_bucket = int(time.time() // 60)
        
        # Aggregate last 60 buckets
        keys = [f"trending:{current_bucket - i}" for i in range(60)]
        
        # Union all buckets into temp key
        temp_key = f"trending:aggregated:{current_bucket}"
        await self.redis.zunionstore(temp_key, keys)
        await self.redis.expire(temp_key, 60)
        
        # Get top N
        return await self.redis.zrevrange(temp_key, 0, limit - 1, withscores=True)
```

## 14. Interview Tips

### Common Follow-up Questions

1. **How do you handle celebrity tweets?**
   - Hybrid fanout (push for regulars, pull for celebrities)
   - Pre-compute for some followers, on-demand for others

2. **How do you ensure timeline consistency?**
   - Eventually consistent is acceptable
   - Use idempotent updates
   - Cache invalidation on critical updates

3. **How do you handle tweet deletion?**
   - Soft delete in DB
   - Async removal from caches
   - Tombstone records for consistency

4. **How do you implement "You might like" recommendations?**
   - Collaborative filtering
   - Content-based similarity
   - Graph-based (friends of friends)

5. **How do you prevent spam?**
   - Rate limiting
   - ML-based spam detection
   - Account verification
   - CAPTCHA for suspicious activity

---

## âœ… Key Takeaways

1. **Hybrid fanout is essential** - Push for regular users, pull for celebrities
2. **Read-heavy workload** - Cache aggressively (timelines, profiles, tweets)
3. **Eventual consistency is OK** - Users accept slight delays in feed
4. **Separate read/write paths** - Optimize each independently
5. **Async processing** - Fanout, search indexing, notifications
6. **Sharding strategy matters** - Tweet ID for writes, user ID for reads
